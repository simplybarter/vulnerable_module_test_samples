# Activity Report Template

Use this template when writing or appending to:

- `logs/activity/activity-[YYYY-MM-DD].md`

Append entries to the daily file rather than creating ad hoc report names.

Each entry must begin with a timestamp in 24-hour format.

```md
## [YYYY-MM-DD HH:MM]

### Language

- `<language>`

### Workflow Assessment

- Current coverage:
- Gap reviewed:
- Why it matters:
- Decision:

### Research Sources

- `<official source 1>`
- `<official source 2>`
- `<advisory source>`

### Vulnerable Package Selection

- Package:
- Version:
- Rationale:

### Implementation

- Files added:
- Files updated:
- Fixture behavior introduced:

### expected.json Assessment

- Ecosystem label:
- Discovery types used:
- Validation notes:

### Documentation Impact

- Language README updates:
- Root documentation updates:
- Template or schema updates:

### Repository Impact

- Scanner behaviors now covered:
- Remaining gaps:
- Risks or follow-up:
```
