# Code of Conduct

This project aims to be a practical, respectful open-source collaboration space.

## Our Standard

Participants are expected to:

- communicate respectfully and stay focused on the work
- assume good intent while still discussing technical disagreements directly
- give actionable, evidence-based feedback
- avoid harassment, discrimination, intimidation, and personal attacks
- keep discussions relevant to the repository and its goals

## Unacceptable Behavior

The following behavior is not acceptable in project spaces:

- abusive or degrading language
- harassment or deliberate intimidation
- trolling, baiting, or repeated bad-faith argument
- publication of private information without permission
- discriminatory conduct based on personal characteristics

## Scope

This code of conduct applies to:

- GitHub issues
- pull requests
- review comments
- project discussions and other repository-managed collaboration spaces

## Reporting

If you experience or witness unacceptable behavior, contact the project maintainers through the channel they designate for repository administration or moderation.

When reporting, include:

- what happened
- where it happened
- links or screenshots if available
- any immediate moderation concern

## Enforcement

Project maintainers may remove comments, close discussions, reject contributions, or restrict participation when behavior violates this code of conduct.

The goal is to protect a workable collaboration environment, not to litigate every disagreement in public.
