# Support

## What This Repository Supports

This repository supports use cases related to dependency-discovery fixture testing, including:

- understanding the fixture layout
- interpreting `expected.json`
- proposing new fixture coverage
- clarifying whether a scanner result matches the repository's intended expectations

## Best Channel

Use GitHub issues for:

- documentation problems
- fixture correctness problems
- requests for additional ecosystem or package-manager coverage
- questions that reveal ambiguity in the repository contract

Use pull requests when you already know the change needed and can propose it directly.

## What Support Does Not Cover

Support for this repository does not include:

- operating a specific commercial scanner
- triaging live vulnerabilities in external projects
- guaranteeing that every dependency-analysis tool will interpret every ecosystem identically
- maintaining private forks or custom fixture variants for downstream users

## Before Opening An Issue

Please check:

- the root [README.md](README.md)
- the language-specific README for the relevant fixture directory
- the fixture's `expected.json`

If the question is actually about a repository security issue, follow [SECURITY.md](SECURITY.md) instead.
